# Gemini Gems — Editing and Updating Your Gems

Gems are not set in stone. You can go back and adjust any Gem at any time — instructions, knowledge files, name, the whole thing.

---

## How to edit a Gem

1. **Open Gemini:** Go to [gemini.google.com](https://gemini.google.com/)
2. **Find your Gem:** Look for the **Gems** section in the left-hand sidebar.
3. **Open the editor:** Click the Gem name, use the three-dot menu (`⋮`) next to it, or hit the pencil icon — all three get you to the Edit view.
4. **Save your changes:** Edits either save automatically or via a **Save** button depending on the current UI version.

---

## What you can change

| Field | What it does |
|:------|:-------------|
| **Name and description** | How the Gem is identified in your list |
| **Instructions (Prompt)** | The core of the Gem — behavior, tone, rules, constraints |
| **Knowledge files** | Upload new PDFs, TXTs, or CSVs / remove outdated ones |
| **Avatar and sharing** | Profile image and visibility settings |

---

## What to watch out for

### No version history
The platform does not track revision history. If you're modifying a complex, well-tuned prompt — save the current version to an external file first. One overwrite and the previous configuration is gone.

### Changes apply to new conversations only
Existing chat threads ran on the old logic. Modified Gems only take effect in conversations started after the update.

### Shared Gems update live
If your Gem has a public share link, any changes you save will immediately reflect for anyone using that link.

### Use desktop for complex edits
Mobile covers the basics, but the desktop browser gives you a cleaner, more manageable editing surface for longer instruction sets.

---

## Before you make major changes

> Duplicate the Gem or copy the full prompt to an external file before editing. It takes thirty seconds and saves you from rebuilding something that was already working.
